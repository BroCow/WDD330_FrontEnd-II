Week 2 Notes

JavaScript Grammer

C-style - similar to C languages

> ASI automatically adds semi-colons, but it is best practice to add them at the end of each statement.
Blocks - Do not need to end with semi-colons {}
Objects - Anything that isn't a primitive data type

Variables

const = can't change value, but not immutable
Underlying data in object can change (mutating object)
const name = {value: 'Chris'}; name.value = 'Carter';
Try to use const to declare most variables - cuts down on bugs
var = don't use it (outdated) let = replaced var as of ES6 (block-scoped)
Global scope variables - minimize the use to reduce chance of errors
Backslash - can be used to escape characters like '
Properties - accessed by using the . notation (such as .length) OR brackets ['']
Methods - accessed by .notation followed by (), perform an action (like toUpperCase())
Template Literals - AWESOME! Using backticks (`) allows for insertion of variables into strings using ${}; ES6 often uses backticks to create ALL strings!!!
Number.isInteger - used to check if number is/isn't and integer (boolean)
0x = hexidecimal / 0o = octal / 0b = binary
Number() is preferred way to change string to number: Number('23');
String() is preferred way to change number to string: String(23);
Always use HARD EQUALITY to avoid potential coercion issues. If number is submitted in form, convert to number using Number('5') === 5.
