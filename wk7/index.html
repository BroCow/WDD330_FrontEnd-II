Call & Apply Methods

The call() method can be used to set the value of this inside a function to an object that is provided as the first argument.

If the function that’s called requires any parameters, these need to be provided as arguments after the first argument, which is always the value of this.

The apply() method works in the same way, except the arguments of the function are provided as an array, even if there is only one argument

Call & Apply - allow generalized functions to be written that are not tied to specific objects by being methods of that object. This gives flexibility over how the functions can be used.

Memoization - If a function takes some time to compute a return value, we can save the result in a cache property. Then if the same argument is used again later, we can return the value from the cache, rather than having to compute the result again.

Immediately Invoked Function Expressions

Immediately Invoked Function Expression – or IIFE: a useful way of performing a task while keeping any variables wrapped up within the scope of the function. This means the global namespace is not polluted with lots of variable names.

Functions that Define and Rewrite Themselves

A function is able to not only call itself, but define itself, and even redefine itself. This is done by assigning an anonymous function to a variable that has the same name as the function.

Code Example:

function party(){
console.log('Wow this is amazing!');
party = function(){
console.log('Been there, got the T-Shirt');
}
}
const beachParty = party; // note that the party function
➥ has not been invoked
beachParty(); // the party() function has now been
➥ redefined, even though it hasn't been called explicitly -- 'Wow this is amazing!'
party();
-- 'Been there, got the T-Shirt'
beachParty(); // but this function hasn't been redefined
-- 'Wow this is amazing!'
beachParty(); // no matter how many times this is called it ➥ will remain the same
-- 'Wow this is amazing!'

Losing Properties - Lazy Definition Pattern and is often used when some initialization code is required the first time it’s invoked. This means the initialization can be done the first time it’s called, then the function can be redefined to what you want it to be for every subsequent invocation.

Recursive Functions

Invokes itself until a certain condition is met.

Instead of waiting for an event to occur, a callback can be created that’s invoked when the event happens. This means that the code is able to run out of order, or asynchronously.

a callback always has to wait for the current execution stack to complete before it’s invoked.

Promises

Promises - represents the future result of an asynchronous operation

The Promise of a Burger Party3 is a brilliant post by Mariko Kosaka that explains the concept of promises by comparing them to ordering a burger!

A promise is created using a constructor function

Example:

const promise = new Promise( (resolve, reject) => {
// initialization code goes here
if (success) {
resolve(value);
} else {
reject(error);
}
});

Once a promise has been settled, the then() method can be used to deal with the outcome.
