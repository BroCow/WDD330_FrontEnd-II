Errors, Exceptions, and Warnings

Usually caused by:

  • System error - problem with system or external devices
  • Programmer error - incorrect syntax or faulty logic (maybe typo)
  • User error - incorrect data entered, program can't handle

System error - not much we can do about it

Programmer error - our responsibility

User error - program apps to prevent this

Exceptions - produces a return used by program to deal with error

Stack Traces - produced by exceptions; works backwards from error to identify original function that started process

Warnings - error in code not severe enough to crash program

The Importance of Testing and Debugging

Ensure code fails LOUDLY during production - not quitely; make sure exceptions are caught and delt with

Test rigorously

Strict Mode

Produces more exceptions and warnings; prohibits used of some deprecated features - considers poor style as errors

Improves clarity and speed

Add to first line of JS: 'use strict';

Per Function - add to first line within function

Preferred Method: Place all code into a self-invoking function:
(function()){
'use strict';
// all code goes here
}());

Modules in strict mode by default

Linting Tools - used for consistent code that is less likely to have problems

Feature Detection - determines browser support for a feature; uses an if statement to check if object or method exists before calling

Modernizr & Can I Use - resources for implementing feature detection

Debugging in Browser

Alert - don't use; old school

Console.log - use to log variables, doesn't stop code

Console.trace - logs interactive stack trace

Debugger keyword - activates browser's debugging tool for use in console; ALWAYS REMOVE BEFORE SHIPPING CODE

Error Objects

Can be created using constructor funtion:
const error = new Error(); // Takes parameter such as Error('Oops, something went wrong');

7 Specific error objects:

  • EvalError - no longer used
  • RangeError - if number outside allowable range of values
  • ReferenceError - reference to item that doesn't exist
  • SyntaxError
  • TypeError - incorrect value type used, like a string instead of number
  • URIError - problem encoding or decoding URL
  • InternalError - error in Javascript engine
