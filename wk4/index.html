Object-Oriented Programming

Three main concepts in OOP:

  • encapsulation
  • polymorphism
  • inheritance

Encapsulation: the inner workings are kept hidden inside the object and only the essential functionalities are exposed to the end user, such as the “on” button.

Polymorphism: the same process can be used for different objects.

Inheritance: taking the features of one object then adding some new features.

Class-based languages - Use a class to define a blueprint for an object. Objects are then created as an instance of that class, and inherit all the properties and methods of the class.

Prototype- based language - Uses the concept of using actual objects as the blueprint for creating more objects.

Constructor Functions

Constructor Function - a function that defines the properties and methods of an object

Easiest way to create a new object is to use the literal syntax

ES6 Class Update - class declaration syntax (see example in wk4.js)

Static Methods

Static Methods - called by the class directly rather than by instances of the class.

Static methods are not available to instances of the class

Prototypal Inheritance

Every class has a prototype property that is shared by every instance of the class.

Modular JavaScript

The code in a module should have a single purpose, and group together functions with distinct functionality.

Keeping code modular helps to make it more loosely coupled and interchangeable, meaning you can easily swap one module for another without affecting other parts of a project.

Coupling - Two pieces of code are said to be tightly coupled if one relies on the other to run. This often occurs if a piece of code makes hard-coded references to another piece of code, requiring it to be used.

ES6 - Added use of modules to JS

  • All code in modules is always in strict mode without the need for 'use strict' and there is no way to opt out of this.
  • A module has its own global scope, so any variables created in the top-level of a module can only be accessed within that module.
  • The value of this in the top level of a module is undefined, rather than the global object.
  • You can’t use HTML-style comments in modules (although this isn’t very common in any JavaScript program these days).

Export keyword - sets functions up for ability to be imported

Importing - can import single functions, or all functions using * and a name that creates a namespace for all the functions

MVC Frameworks

Models

  • CRUD tasks - creating, reading, updating, deleting

Views

  • Visual representation of the model
  • Provides user interaction - usually through forms

Controllers

  • Link models and views together
  • Respond to events - usually inputs from users
  • Process info and update model/view
  • Respond to event from user, then inform model task completed - model then updates info about that task
