Week 10 Notes

MDN - Client-side Form Validation

Makes for good user experience, but does not provide all the protection necessary

What is form validation?

Form Validation - Restrictions on the type and format of data entered by user

Client-side: Done in browser

Server-side: Done at server

Reasons for form validation:

  • We want to get the right data, in the right format
  • We want to protect our users data
  • We want to protect ourselves

Never trust data entered by a user!!!

Different types of client-side validation

Built-in form validation - uses HTML5

JavaScript - coded and customizable

Using built-in form validation

  • Required - specifies if field must be completed
  • minlength / maxlength - how long the text string
  • min / max - specifies value requirement of numbers
  • Type - type of entry(email, phone, text, etc.)
  • Pattern - sets pattern of data required(uppercase, lowercase, numbers)

CSS :valid - activated when entry acceptable

CSS :invalid - activated when entry not acceptable

The Constraint Validation API

Methods and properties available for:

  • HTMLButtonElement - button elements
  • HTMLFieldSetElement - fieldset
  • HTMLInputElement - input
  • HTMLOutputElement - output
  • HTMLSelectElement
  • HTMLTextAreaElement

Available properties

  • validationMessage
  • validity
  • willValidate

Available methods

  • checkValidity()
  • setCustomValidity(message)

Common Use - creating custom error messages (content and appearance)

Novalidate form - stops browser from displaying its own message
